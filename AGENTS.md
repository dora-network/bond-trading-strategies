# AGENTS.md — bond-trading-strategies

## Three binaries

```
cmd/mcp-server/       — MCP HTTP/SSE server (:8080)
cmd/strategy-server/  — REST strategy API (:8081)
cmd/price-daemon/     — WebSocket market data ingester (:8080 health)
```

## Start locally (from project root)

```bash
make start-strategy-server   # requires DATABASE_URL + DORA_API_KEY
make start-mcp-server        # requires DORA_API_KEY, STRATEGY_BASE_URL
make start-price-daemon      # requires DATABASE_URL + DORA_API_KEY
```

Order: price-daemon → strategy-server → mcp-server. `.env` is auto-sourced by Makefile.

## Database

- PostgreSQL with `tern` migrations in `migrations/`
- Run: `tern migrate --config migrations/tern.conf`
- Schema: `price_history`, `candles_history`, `strategy_runs`, `strategy_backtests`

## Tests

```bash
go test ./...                # all tests
go test -race ./strategy/... # race-detector
go test ./mcp/...            # MCP server tests
go test -run TestFoo ./...   # single test
```

Uses `testify` (assert/require). Fakes generated by `counterfeiter` (see `//go:generate` in `strategy/service.go:15` and `strategy/strategy.go:12`). MCP tests use `mcptest` package. White-box test helpers via `export_test.go` (strategy, meanreversion, candles, prices).

## Auth

Every strategy-server endpoint except `/healthz` requires `Authorization: ApiKey <key>` or `Bearer <token>`. The DORA API key is validated against the live DORA API. MCP server requires `DORA_API_KEY` env var to start.

## Architecture notes

- `strategy.Service` manages backtest/run lifecycle with `context.CancelFunc` maps + channel-based messaging (`Stop`, `Pause`, `Resume`)
- `streams.Daemon` wraps any `StreamFunc` with auto-reconnect (applies to both prices and candles)
- `strategy/http.RunStore` / `BacktestStore` interfaces have PG implementations with upsert semantics
- `strategy/http.Handler` serves as both the REST handler and the run/backtest state manager
- `price-daemon` uses functional options (`WithMessageHook`, `WithWriteHook`) for health-check integration
- MCP server proxies strategy requests to `strategy-server` over HTTP; FRED tools call FRED API directly
- All monetary values use `github.com/govalues/decimal` — never `float64`
